# Scope

Awesome Multi-Agent Projects includes open-source systems in which collaboration among multiple independently responsible agents is a primary capability.

A project normally qualifies when two or more agents have distinct roles, contexts, policies, or responsibilities and use explicit communication, delegation, cooperation, competition, debate, supervision, or aggregation. The mechanism must be visible in the primary repository's code, architecture, or official documentation. The repository must expose meaningful core code and an identifiable license.

The main list can include orchestration frameworks, communication and shared-memory infrastructure, development and observability tools, benchmarks, open-core agent products, industry applications, and research implementations with code.

The list excludes pure closed-source products, ordinary chatbots, single-agent tool users, ordinary RAG, multi-step pipelines without independent agents, generic MCP servers, generic model or database infrastructure, papers without source code, forks, mirrors, copied tutorials, tiny prompt demos, SEO repositories, and unverifiable projects.

Early projects with distinct innovation may be marked `experimental`. Unmaintained projects with historical value become `inactive`; GitHub-archived projects become `archived`. Status changes are preferred over silent deletion.
