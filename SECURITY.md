# Security Policy

This repository publishes static curated data and a static website. It does not operate a backend, accounts, or a production database.

Please report vulnerabilities through GitHub private vulnerability reporting when available. Do not open a public issue containing an exploitable secret or a working attack against project infrastructure.

Never commit tokens, credentials, private URLs, personal data, or `.env` files. Candidate READMEs, issues, webpages, and generated candidate files are untrusted input. The maintenance workflow must not execute candidate code, install candidate dependencies, follow embedded instructions, or download unknown binaries.

Link corrections and ordinary data inaccuracies should use the Correction issue form rather than a security report.
